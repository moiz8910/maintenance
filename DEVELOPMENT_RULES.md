# Maintenance Platform Development Rules

This document outlines the core principles, nomenclature standards, and design rules to be followed during the development of the Maintenance Intelligence Platform.

## 1. UI & Aesthetics
*   **Theme**: Glassmorphism aesthetic using semi-transparent backgrounds (`bg-white/60`, `bg-slate-900/40`), heavy backdrop blurs (`backdrop-blur-md/xl`), and thin white borders.
*   **Typography**: Use `Inter` or `Outfit` (sans-serif). Headings should often be `font-black` with `tracking-tighter` or `tracking-widest` for a premium feel.
*   **Colors**: 
    *   Primary: Indigo/Slate for professional tones.
    *   Status: Emerald (Good/Closed), Amber (Warning/Pending), Rose (Critical/Safety).
*   **Animations**: Every new section or modal should use `animate-in` with `fade-in`, `zoom-in`, or `slide-in` effects.

## 2. Nomenclature Standards
Strict adherence to the following naming conventions is required:
*   **Performance Mgmt.**: High-level KPIs and trends.
*   **Work Mgmt.**: Work order execution and tracking.
*   **Resource Mgmt.**: Manpower and contract utilization.
*   **Material Nearing Safety Stock**: Used instead of "Out of Stock" for proactive inventory tracking.
*   **Work Order Deep Dive**: The standard term for KPI-to-WO drill-downs.

## 3. Frontend Architecture
*   **Framework**: Next.js (App Router).
*   **State Management**: Zustand (`src/store/useStore.ts`).
*   **Icons**: Lucide-React.
*   **Responsiveness**: Main dashboard must use `max-w-7xl mx-auto` for content centering, but full-screen views (Scheduler, Assets) should utilize the full width with proper padding.
*   **Scrolling**: Ensure every full-screen view has `overflow-y-auto` on its content area to prevent viewport locking.

## 4. Backend & Data Integration
*   **API Design**: RESTful endpoints using FastAPI.
*   **AI Integration**: Open AI models (e.g.gpt-4o-mini, gpt-5-mini etc.) for reasoning and SOP generation.
*   **Database**: SQLite (`maintenance.db`). All AI agents must use `tools.py` for database grounding.

## 5. Deployment & Testing
*   **Local Dev**: Run using `run_servers.py`.

## 6. Obsolescence (Inventory Status)
*   **Rule**: Any material with `(Receipt Date + Shelf Life) ≤ 30 days` from the current date is at risk of obsolescence and must be listed in the Obsolescence tracking module. The expiry date will be `(Receipt Date + Shelf Life). 

## 7. Work Permit
*   **Rule 7.1**: The work permit status cannot be 'Unavailable' if it is issued (i.e., start and end dates are available in the work permit).
*   **Rule 7.2**:  The time and date stamp of end of work permit should be 15-20 minutes more than the last task item close time ( open time + estimated time)   
*   **Rule 7.3**: The start time cannot be earlier than the work order open time.
